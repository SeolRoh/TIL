#### Kubernetis 배포

---

주요개념

+ Node

  :컨테이너가 배치되는 서버

+ Namespace

  :쿠버네티스 클러스터 안의 가상 클러스터

+ Pod

  :컨테이너의 집합 중 가장 작은 단뒤, 컨테이너의 실행 방법 정의

+ Repica Set

  :같은 스펙을 갖는 파드를 여러 개 생성하고 관리하는 역할

+ Deployment

  :레플리카 세트의 리비전을 관리

+ Service

  :파드의 집합에 접근하기 위한 경로를 정의

+ Ingress

  :서비스를 쿠버네티스 클러스터 외부로 노출

+ ConfigMap

  :설정 정보를 정의하고 파드에 전달

+ Persistent Volume

  :파드가 사용할 스토리지의 크기 및 종류를 정의

+ Persistent Volume Claim

  :퍼시스턴트 볼륨을 동적을 확보
